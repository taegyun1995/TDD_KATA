# Java 로또(Lotto) 게임

TDD와 단위 테스트 학습을 위한 간단한 콘솔 기반의 로또 게임 애플리케이션입니다.

## 🎯 기본 기능 요구사항

### 1. 로또 구입
- 사용자는 로또 구입 금액을 입력한다.
- 로또 1장의 가격은 1,000원이다.
- 구입 금액은 1,000원 단위로 입력받으며, 그렇지 않을 경우 예외를 발생시킨다.
- 입력된 금액만큼 로또를 **자동으로** 발행해야 한다.

### 2. 자동 로또 번호 생성
- 로또 1장은 1부터 45 사이의 중복되지 않는 6개의 숫자로 구성된다.
- 발행된 각 로또의 번호는 오름차순으로 정렬하여 보여준다.

### 3. 당첨 번호 및 보너스 번호 입력
- 사용자는 당첨 번호 6개와 보너스 번호 1개를 입력한다.
- 각 번호는 1부터 45 사이의 값이어야 하며, 서로 중복될 수 없다.

### 4. 당첨 통계
- 사용자가 구매한 모든 로또를 당첨 번호와 비교하여 당첨 결과를 결정한다.
- 당첨 기준은 다음과 같다.
    - 1등: 당첨 번호 6개 모두 일치 (상금: 2,000,000,000원)
    - 2등: 당첨 번호 5개 + 보너스 번호 일치 (상금: 30,000,000원)
    - 3등: 당첨 번호 5개 일치 (상금: 1,500,000원)
    - 4등: 당첨 번호 4개 일치 (상금: 50,000원)
    - 5등: 당첨 번호 3개 일치 (상금: 5,000원)
- 각 등수별 당첨 개수와 총 수익률을 계산하여 출력한다.

---

## 🚀 심화 기능 요구사항 (선택)

### 5. 수동 로또 번호 구매
- 로또 구매 시, 자동 구매뿐만 아니라 사용자가 직접 번호를 입력하는 **수동 구매** 기능을 추가한다.
- 예를 들어, 3장을 구매할 경우 1장은 수동, 2장은 자동으로 구매할 수 있다.
- 수동으로 입력한 번호가 유효한지(6개인지, 1~45 범위 내인지, 중복은 없는지) 검증하는 로직을 추가하고 테스트한다.

---

## ✅ 프로그래밍 요구사항
- 모든 기능은 TDD(Test-Driven Development) 방식으로 개발한다.
- `Enum`을 활용하여 당첨 등수(상금, 일치 개수)를 관리한다.
- 컬렉션(List, Set)을 효과적으로 사용하여 로또 번호를 관리하고 테스트한다.
- 테스트 라이브러리는 **JUnit 5**와 **AssertJ**를 사용한다.
