# Java 문자열 계산기

TDD 사이클(Red-Green-Refactor)을 연습하기 위한 고전적인 TDD Kata(카타)입니다.
하나의 클래스, 하나의 메소드에 점진적으로 기능을 추가하며 TDD를 깊이 있게 훈련하는 것을 목표로 합니다.

## 📚 학습 목표
- TDD의 Red-Green-Refactor 사이클 체득
- 점진적으로 기능을 확장하는 방법 학습
- 문자열 파싱 및 정규 표현식 활용
- 예외 처리 테스트 작성법

---

## 🎯 기능 요구사항

아래 요구사항을 한 단계씩 순서대로 TDD 사이클에 맞춰 개발합니다.

### Step 1. 빈 문자열 또는 null 처리
- `add(String text)` 메소드에 빈 문자열이나 `null`을 전달할 경우, `0`을 반환한다.

### Step 2. 숫자 하나
- 숫자 하나를 문자열로 전달할 경우, 해당 숫자를 반환한다. (예: `"1"` -> `1`)

### Step 3. 쉼표(`,`) 구분자
- 쉼표를 구분자로 사용하여 숫자들의 합을 구한다. (예: `"1,2"` -> `3`, `"1,2,3"` -> `6`)

### Step 4. 줄바꿈(`\n`) 구분자
- 쉼표 외에 줄바꿈 문자도 구분자로 사용할 수 있다. (예: `"1\n2,3"` -> `6`)

### Step 5. 커스텀 구분자
- 문자열의 시작이 `//`와 `\n` 사이의 문자를 커스텀 구분자로 사용한다.
- 예: `"//;\n1;2;3"` -> `6` (세미콜론(`;`)이 구분자)

### Step 6. 음수 예외 처리
- 문자열에 음수가 포함되어 있을 경우, `RuntimeException` 예외를 발생시킨다.
- 예외 메시지에는 발견된 모든 음수 목록을 포함해야 한다. (예: `"-1,2,-3"` -> `RuntimeException("음수는 허용되지 않습니다: -1, -3")`)

### Step 7. 숫자가 아닌 값 예외 처리
- 구분자로 분리한 값 중 숫자가 아닌 값이 있을 경우, `RuntimeException` 예외를 발생시킨다.
- 예: `"1,a,3"` -> `RuntimeException("숫자가 아닌 값이 포함되어 있습니다: a")`

## 🚀 심화 기능 요구사항 (선택)

### Step 8. 1000 초과 숫자 무시
- 덧셈을 할 때 `1000`을 초과하는 숫자는 무시하고 더한다.
- 예: `"2,1001"` -> `2` (1001은 무시)

### Step 9. 여러 커스텀 구분자
- 대괄호(`[]`)를 사용하여 여러 개의 커스텀 구분자를 지정할 수 있다.
- 예: `"//[*][%]\n1*2%3"` -> `6`

### Step 10. 뺄셈 연산
- `subtract(String text)` 메소드를 추가한다.
- 첫 번째 숫자에서 나머지 숫자들을 뺀 결과를 반환한다.
- 예: `"10,3,2"` -> `5` (10 - 3 - 2)

## ✅ 프로그래밍 요구사항
- 모든 기능은 TDD(Test-Driven Development) 방식으로 개발한다.
- **한 번에 하나의 테스트 케이스만 통과시키고, 다음 단계로 넘어간다.**
- `String` 클래스의 메소드와 정규 표현식을 적극적으로 활용한다.
- 테스트 라이브러리는 **JUnit 5**와 **AssertJ**를 사용한다.
