# Java 카페 주문 시스템

TDD와 단위 테스트 학습을 위한 간단한 콘솔 기반의 카페 주문 시스템입니다.

## 🎯 기본 기능 요구사항

### 1. 메뉴판
- 시스템은 시작 시 메뉴 목록을 가지고 있어야 한다. (예: 아메리카노-4000원, 라떼-4500원, 카푸치노-5000원)
- 메뉴는 이름과 가격 정보를 가진다.

### 2. 주문하기
- 사용자는 메뉴판에 있는 메뉴를 이름으로 주문할 수 있다.
- 주문 시 수량을 함께 입력할 수 있다. (예: "아메리카노 2개")
- 주문은 여러 개의 메뉴 항목(메뉴와 수량)으로 구성될 수 있다.

### 3. 주문 내역 및 총액 계산
- 현재까지 주문한 모든 메뉴 항목과 각 항목의 가격, 총 주문 금액을 계산하여 보여줄 수 있어야 한다.

### 4. 결제
- 사용자가 특정 금액을 지불하여 결제를 시도할 수 있다.
- 지불한 금액이 총 주문 금액보다 적을 경우, "금액이 부족합니다." 라는 예외를 발생시킨다.
- 결제가 성공하면, 지불한 금액에서 총 주문 금액을 뺀 거스름돈을 계산하여 반환한다.

---

## 🚀 심화 기능 요구사항 (선택)

### 5. 재고 관리
- 각 메뉴별로 재고 수량을 관리한다.
- 주문이 들어올 때마다 해당 메뉴의 재고가 차감된다.
- 주문하려는 수량보다 재고가 부족할 경우, "재고가 부족합니다." 라는 예외를 발생시킨다.

### 6. 할인 정책 추가
- 다양한 할인 정책을 적용할 수 있도록 시스템을 확장한다.
- 예시:
    - **금액 기반 할인:** 총 주문 금액이 20,000원 이상이면 10% 할인
    - **특정 메뉴 할인:** 아메리카노 메뉴에 대해 500원 할인
- 여러 할인 정책이 중복으로 적용될 수 있도록 설계한다.

---

## ✅ 프로그래밍 요구사항
- 모든 기능은 TDD(Test-Driven Development) 방식으로 개발한다.
- 모든 기능 단위에 대한 단위 테스트를 작성한다.
- Java 코드 컨벤션을 지키면서 프로그래밍한다.
- 객체의 역할과 책임을 명확하게 분리한다. (예: `MenuItem`, `Order`, `Cashier`, `Inventory`, `DiscountPolicy`)
- 테스트 라이브러리는 **JUnit 5**와 **AssertJ**를 사용한다.
